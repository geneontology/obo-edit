Contructor for the bbop.golr.manager.widget.autocomplete object.

Yes, that is very long.

This is a specialized (and widgetized) subclass of bbop.golr.manager.

The function for the callback argument should either accept a JSONized solr document representing the selected item or null (nothing found).

Arguments

golr_locstring url to GOlr server;
golr_conf_obja bbop.golr.conf object
interface_idstring id of the element to build on
output_type[optional] thing to put in the input box: ‘id’ or ‘label’
select_callback[serially optional] function takes a jsonized solr doc

Returns

this object

make_active

this.make_active = function()

Makes the autocomplete active.  This must be done after everything has been loaded (e.g. onload or whatever for what you’re using) or else serious weirdness happens.

Again, since this is adding jQuery listeners to a finished document--it must be done “late”.
