Rerooting

Works by: Edit menu, right-click

Hot keys: none

Source requirements: One source relationships required - see below for special requirements for the source relationship

Target requirement: No target required

Occasionally, it may be necessary to transform a non-root term into a root term. This can be done using the "Reroot" operation.

To reroot a term, first make sure it has only one parent relationship. (If it has others, Delete them before rerooting). Select the term to reroot, and choose the "Reroot" option from the right-click or edit menu.

The term will be removed from its remaining parent and will appear as a root term.
