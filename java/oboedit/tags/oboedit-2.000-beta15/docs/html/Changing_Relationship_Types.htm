Changing Relationship Types

Works by: Drag-and-drop, right-click, edit menu

Hot keys: T

Source requirements: One or more source relationships allowed for right-click and edit menu, a single source property required for drag-and-drop

Target requirement: One target required for drag-and-drop, no target required for right-click or edit menu

Once a relationship has been created between two terms, the relationship type (aka property) of the relationship can be changed.

Using the right-click and edit menus

Select one or more source relationships to change. Choose "Change relationship type" from the right-click menu or edit menu. A list of possible relationship types will appear:

Choose a new relationship type from the list. All source relationships will be changed to use the new relationship type. Note that some relationship types will not be available because of domain and range restrictions, or because the selected relationship already uses that type.

Using drag and drop

Drag and drop works very differently than the right-click and edit menus when changing relationship types. To change a relationship type using drag and drop, first select the source property you would like to use. Drag the source property onto the target relationship you would like to change. A popup menu will appear:

Choose "Change relationship type" from the menu (most of the time, this will be the only option available). The target relationship will be changed to use the source relationship type.

Using hotkeys

When dragging, you can hold down the "T" key to specify the "Change relationship type" operation. The title above the terms you are dragging will change from "Dragging" to "Move...":

When the source relationship type is dropped, the target relationship will immediately change to use the new relationship type.
