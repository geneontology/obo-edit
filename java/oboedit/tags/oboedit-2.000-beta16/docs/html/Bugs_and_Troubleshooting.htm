Bugs and Troubleshooting

When installing a new version of OBO-Edit, check the README file for a summary of changes in the new version, any known issues, and any last-minute release notes or documentation.

Bug Reports

OBO-Edit is a work in progress. Bugs can and will creep in. Please report any odd behavior or bugs immediately. The OBO-Edit Working Group is very responsive to bug reports, and your bugs will be addressed within a few days (or sometimes hours).

To report a bug, please use our Sourceforge bug tracker at:

http://sourceforge.net/tracker/?group_id=36855&atid=418257

Feature Requests

We're always listening to suggestions for new OBO-Edit features. To request a new feature, please use our Sourceforge Feature Request Tracker at:

http://sourceforge.net/tracker/?group_id=36855&atid=418260

Other Requests & Last Resort Bug Reporting

If you have a special problem that needs immediate attention, please email John Day-Richter (the lead OBO-Edit developer) at jmr39@users.sourceforge.net. He will try to answer your email as soon as possible.

You may also want to try visiting the #oboedit channel on irc.sf.net. John and other working group members occasionally monitor this channel.
