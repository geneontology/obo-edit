The OWL Adapter

Overview

What it does: Reads and writes files in the OWL format. The adapter converts OWL to OBO and OBO to OWL using the oboformat library.

Operation

Basic Operation

The OWL adapter's basic mode is similar to the interface described in The Basic Flat File Interface.

Advanced Operation

Loading

To the right of the Data adapter pulldown menu, there's a button that (when you're in Basic mode) says "Advanced". When the "Advanced" button is pressed, the load dialog changes to look like this:

This interface allows fine control over which files are loaded. The list to the left contains all the files to be loaded by the OWL Adapter.

To edit a filename, click the filename to select it. It will be loaded into a text box to the right:

To change the path, type a new path into the text box, or select another file from your computer by clicking the "Browse..." button.

To add a new file path, press the "Add" button. A new path will appear with the path set to "<new path>". Edit the path as described above.

To delete a path, select it from the list and press the "Del" button.

The "Allow dangling references" box enables the OWL Adapter to load converted ontologies that contain dangling references after the conversion from OWL. The behavior for dangling reference is the same as for the OBO Adapter.

Saving

Saving in "Basic" mode is straightforward. The Adapter writes a temporary OBO file and converts it to OWL. Both, simple and advanced mode, do not support multiple output files or profiles.

One know bug (on MacOSX 10.6) is that the file dialog sometimes removes the suffix of the output File. Please check that suffix is properly appended to the write path.
