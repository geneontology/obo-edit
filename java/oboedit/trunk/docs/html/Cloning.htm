Cloning

Works by: Edit menu, right-click

Hot keys: Command+L

Source requirements: One or more source relationships required

Target requirement: No target required

The Clone operation creates an exact duplicate of one or more terms, but assigns new id numbers to the duplicates. The duplicate terms have the same parents, children, namespace, definition, comments, synonyms, dbxrefs, and subset assignments as the original term (alternate ids are not preserved). The name of the cloned term is "CLONE OF <the original term name>".

To clone one or more terms, select the terms and choose "Clone" from the right-click or edit menu, or press the "Command" and "L" keys.

Note that the Clone operation will give the cloned terms exactly the same relationships as the original source terms. If parent-child relationships exist between the source terms, parallel parent-child relationships between the cloned terms will not be created.
