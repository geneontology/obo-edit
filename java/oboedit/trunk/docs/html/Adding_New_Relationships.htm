Adding New Relationships

Works by: Drag-and-drop, right-click

Hotkeys: Command+C, Command+P

Source requirements: One or more source relationships required

Target requirement: One target required

New relationships can be added by drag-and-drop, or via the right-click menu.

To create one or more new relationships, select a collection of source relationships that you would like to relate to a new term. Then, select a target term to which they will be related by right-clicking the target term or by drag-and-drop.

From the right-click menu or the drag and drop popup menu, you may add new relationships using the "Add parent term" or "Add child term" menu options. Note that the names of these operations refer to changes that will occur to the target term. So if you drag term A onto term B and choose "Add parent term", A will become a new parent of B.

Both the "Add parent term" and "Add child term" menu items have a flyout submenu of the relationship type that should be used when creating the new relationship:

Select the relationship type you would like for the new relationship, and it will be created. If you would like the new relationship to use the same relationship type as the original source relationship, use the "Use original type" menu option. This is particularly useful if you are creating several relationships at once by using a number of source relationships. If "Use original type" is used with untyped source relationships (ie one or more of your sources is an ontology root) the is_a relationship is used.

Why is my new relationship not showing up in OBO-Edit?

OBO-Edit can display any relationships that are specified in an OBO file, but by default it only shows transitive relationships. If you are not seeing your new relationships, then it's likely that they are not marked as transitive in your file. You have two options:

  • If the relationship is transitive, add the line "is_transitive: true" in the [Typedef] stanza in the file, or select the relationship type in OBO-Edit (under "Relations", in blue), tick the "Is transitive?" box, and save the file.
  • If the relationship is not transitive, you will have to remove one of the default Global Filters. If it's not already open, open the Global Filter component ([View] -> [Show] -> [Config] -> [Global Filters]) and remove the link filter "Link type has Is Transitive". The relationship should become visible when you click "Save changes" (note that if you are using one of the OBO-Edit 2.0 betas, you may have to quit and restart, due to a bug).

Hotkeys

When dragging, you can use hotkeys to specify the "Add parent term" or "Add child term" edit operations. To specify "Add child term", press the "Command" and "C" keys on the keyboard, then start dragging. The hotkey dropdown menu will change to "Add child term (copy)" and the title above the terms you are dragging will change from "Dragging" to "Add child...":

When the term is dropped, the type selection menu will pop up.

To specify "Add parent term", press the "Command" and "P" keys on the keyboard, then start dragging. The hotkey dropdown menu will change to "Add parent term" and title above the terms you are dragging will change from "Dragging" to "Add parent..."

You can also use hotkeys to choose which relationship type will be assigned when you drop the dragged terms. Each relationship type is assigned a hotkey (Command+another key, usually just the number that corresponds to the relationship type's position in the list, but IS_A always gets the hotkey "I").

 

Where did the "copy" menu item go?

Long-time users of DAG-Edit might be wondering where the "Copy" menu item went. The "Add parent term" and "Add child term" menu items have replaced "copy". In fact, "Add child term" does exactly the same thing as the old "Copy" menu item.

We originally called this operation "Copy" because it made a term appear in the graph more than once; a "copy" was created. Over time, we realized that "Copy" was a misleading name. Many new users of DAG-Edit thought that an entirely new term was being generated.
