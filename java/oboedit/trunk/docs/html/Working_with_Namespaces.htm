Working with Namespaces

Every OBO ontology contains a set of namespaces. Terms and relationships in the ontology can be assigned to one of the namespaces in that set (see An Introduction to OBO Ontologies, Text Editing, and The Parent Editor for more information on how to assign entities to namespaces).

The built-in Namespace Manager allows users to manage the set of available namespaces. To access the Namespace Manager, choose "Namespace Manager" from the menu under [View]->[Show]->[Metadata].

Committing Changes

Any changes made in the Namespace Manager are not saved until the "Save Changes" button is clicked. If changes are made and the window is closed, the changes will not be saved. To revert to the last committed version of the namespace set, click the "Revert" button.

Editing An Existing Namespace

To edit an existing namespace, click the namespace to edit in the list on the left. The namespace editor panel will appear. Namespaces have only one editable attribute: their id. To change the namespace id, click the namespace id in the text field and type a new id.

Deleting an Existing Namespace

To delete an existing namespace, click the namespace to delete in the list on the left. Click the Del button to remove the namespace.

Note that a namespace that is assigned to a term or relationship cannot be deleted. If a namespace is deleted that is still in use, a warning will appear when the "Save Changes" button is pressed. If a namespace must be deleted, first search the ontology to make sure that the namespace is not in use.

Adding a New Namespace

To add a new namespace, click the "Add" button. A new namespace will be created with the default id "<new namespace id>". Edit the id as described above.
