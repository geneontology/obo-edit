Getting OBO-Edit Started

OBO-Edit is a Java application, so you will need to have Java on your computer. Most new computers come with some version of Java already installed.

What version of Java do I have?

To figure out which version of Java you have, open up a command prompt, or console, or terminal, or whatever you use on your platform to type commands onto a command-line. Then type:


java -version

and press enter. You should get a response like:


java version "1.5.0_06"
Java(TM) 2 Runtime Environment, Standard Edition (build 1.5.0_06-b05)
Java HotSpot(TM) Client VM (build 1.5.0_06-b05, mixed mode, sharing)

The very first line tells you which version of Java is installed. Make sure the version number is 1.5.something or 1.6.something.

If you get a response like:


'java' is not recognized as an internal or external command, operable program or batch file.

or


bash: java: command not found

check that you typed a space after "java" and before "-version". If you entered the command correctly but still do not get a response indicating a specific version of Java, then Java is not installed, and you need to follow the installation instructions below.

1) Install Java 1.5 or higher

On Macs, the latest version of Java can be installed automatically by doing a system software update. On any other platform, download the appropriate Java Runtime Environment (JRE) binary from Oracle. (The latest version, as of 2011, is Java SE 6.)

2) Get OBO-Edit

Download the latest version of OBO-Edit from our SourceForge project at http://sourceforge.net/projects/geneontology

There will be several downloads to choose from (Windows, Mac, Linux/Unix).

3) Install and Launch OBO-Edit

Memory allocation

Because OBO-Edit2 is a Java application, it cannot use an arbitrary amount of system memory like a native application. Instead, you must set a fixed maximum amount of memory that OBO-Edit2 can use. Unless you have a 64-bit Java Virtual Machine (JVM), Java limits the maximum memory allocation to 1860M (1024M on Windows machines). The OBO-Edit installer will suggest an appropriate memory setting for your computer. You can change the memory setting after installation via OBO-Edit's Configuration Manager or by hand-editing oboedit.vmoptions (located in the OBO-Edit installation directory).

For MacOSX there are three requirements to use more than 1860M of main memory for Java:

  1. Intel CPU with support for 64-bit
  2. MacOS in 64-bit mode

    This is the default for Lion, but on some versions of Snow Leopard, you may need to change this by hand. Here is a guide to getting your 64-bit-capable Mac to boot into 64-bit mode.

    (Sometimes this step is not required to use Java in 64-mode. But OBO-Edit always checks this feature to prevent a too high memory setting)

  3. 64-bit Java as default

    To change this, open /Applications/Utilities/Java Preferences and make sure that a 64-bit version of Java is selected as the default for Java Applications.

In some cases, it might be necessary to re-install OBO-Edit, to take advantage of the changed default Java.

  • Run the installer program and follow the onscreen prompts. Icons for running/uninstalling the program will be created on your start menu (or desktop). On Macs, if you want an OBO-Edit icon in your Dock, let the installer create a desktop icon and then drag it into your Dock.
  • If you prefer to launch OBO-Edit from the shell (on Unix or Mac), you can use the oe-command-line script. You can make it automatically load an OBO-format file by giving the filename as the command-line argument:
    /Applications/OBO-Edit2/oe-command-line /blahblahblah/foo.obo

Stale files in your configuration directory

If you had already installed a previous version of OBO-Edit2, your configuration directory may contain stale configuration files that clash with the new version, which can cause problems when you try to run the new OBO-Edit2. For best results, remove your configuration directory before launching the new OBO-Edit2. You can find the name of the directory by opening the Configuration Manager.
