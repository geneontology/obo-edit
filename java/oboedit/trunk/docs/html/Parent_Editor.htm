The Parent Editor

The Parent Editor displays the immediate parents of the current sub-selection term:

The name and ID of the sub-selected term appear at the top of the panel. Each additional line in the display represents a parent link from the currently selected term to another term. The fields in each line are:

  • Delete button - Press this button (with trashcan icon) to delete this parent link. (Note that this option is not available if a term has only one parent; to delete the last parent from a term, thereby making the term obsolete, use the Ontology Tree Editor.
  • Relationship type - This is a dropdown list of all the relationship types in the ontology. The current relationship type for this link is selected. To change the relationship type of this link, choose a new item from the dropdown list.
  • Parent term - This is the id and name of the parent term of this link. Click the id to select the parent term.
  • Cardinality control - Pops up a separate window to let you set the cardinality of this link. This information is not currently used.
  • Namespace control - This is a list of all the namespaces in the ontology. This current namespace for this link is selected. To change the namespace of this link, select a new namespace from the dropdown.
  • Show implied links - Checking this checkbox will make implied parents appear in the editor; if the box is not checked, only asserted parents appear. Note that the reasoner must be running for this option to work.

To add a new parent link to the selected term, drag the parent term onto the "Local-select other terms and drop here to add new parents" panel. A new link will be added with the dropped term as the parent, and is_a as the relationship type. Change the relationship type as needed.

Note that you have to drag the desired new parent from an Ontology Tree Editor that is in local selection mode, because you don't want the global selection to change. The most convenient way to do this is to have two Ontology Tree Editor panels open, with one in global and the other in local mode. Select the child in the global tree editor panel (which makes it the selected term in the Parent Editor), then drag the desired new parent from the local tree editor panel into the Parent Editor. To add a parent using a single Ontology Tree Editor panel, select the child with the OTE in global mode, then change to local mode before dragging the new parent.
