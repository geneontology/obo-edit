The Reasoner and the Interface

(This page discusses what a reasoner is.)

You can turn on the reasoner in OBO-Edit by showing the Reasoner Manager ([View] -> [Show] -> [Reasoner] -> [Reasoner Manager]) and selecting a reasoner from the pulldown list. (Once you make your choice, you can get rid of the Reasoner Manager panel and the reasoner selection will stay as you left it.)

OBO-Edit offers a choice of reasoners. The current options are ForwardChainingReasoner and LinkPileReasoner. ForwardChainingReasoner, which was the default reasoner in OBO-Edit before March 2008, is faster but not as complete, and will be phased out. LinkPileReasoner is currently very slow, but more complete.

When one of the reasoners is turned on, the titlebar of the OBO-Edit window says "(REASONER ON)" as a visual reminder. The reasoner causes some operations in OBO-Edit to take longer, so you may want to turn it off if you don't need it.

It's easy to imagine how confusing this would get with an ontology of any complexity.

To reduce this complexity, OBO-Edit trims the ontology before displaying it. Trimming hides links that would unhelpfully clutter the display. The reasoner still knows about these links, but they are not shown to the user.

A link L is trimmed if it satisfies either of the following conditions:

  • All evidence for L is based on the transitivity rule
  • L's child term has any grandparent link with the same relationship type and parent term as L

There is one exception: user-created links are never trimmed. This means that redundant links are always visible, because all redundant links are user-created.
