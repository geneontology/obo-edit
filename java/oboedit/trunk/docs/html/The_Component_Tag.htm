The <component> Tag

The component tag adds an OBO-Edit-specific component to the layout. This is the piece of a layout that does the real work.

Parameters

Name Description Legal Values Default
id

An identifier that indicates which component to display (see below). This parameter is required.

any valid identifier

none

Also see common parameters .

Contents

The component tag may be empty, or may specify the layout of the component. In this sub-layout, component tags may only refer to sub-components of the current component. See sub-components below.

Component Identifiers

The following identifiers are valid in OBO-Edit (note that external plugins may add additional identifiers). Below each identifier are the valid sublayout identifiers for that component:

  • DAG - Creates an Ontology Editor Panel
  • TEXTEDIT - Creates a Text Editor
    • ID_EDITOR - Creates a labeled component that displays the current id
      • id_label - Creates a label that displays the current id
      • secondary_id_label - Creates a label that displays the current term's secondary ids
    • NAME_EDITOR - Creates a labeled name editing component
      • field - Creates a name editor field
    • NAMESPACE_EDITOR - Creates a labeled namespace editor dropdown
      • list - Creates a namespace editor dropdown
    • RANGE_EDITOR - Creates a labeled range editing button
      • range_button - Creates the range editing button
    • DOMAIN_EDITOR - Creates a labeled domain editing button
      • domain_button - Creates the domain editing button
    • PROPERTY_BOXES_EDITOR - Creates a panel containing all the property editing components
      • transitivity_checkbox - Creates the transitivity editor checkbox
      • symmetry_checkbox - Creates the symmetry editor checkbox
      • cyclical_checkbox - Creates the cyclicity editing checkbox
    • DEFINITION_EDITOR - Creates a labeled definition editor
      • field - Creates the definition editor field
      • ref_list - Creates the definition dbxref list editor
      • ref_button - Creates the button that opens the definition dbxref editor window
    • COMMENT_EDITOR - Creates a labeled comment editor
      • field - Creates the comment editor field
    • COMPLETE_DEF_EDITOR - Creates a labeled cross product editor
    • DBXREF_EDITOR - Creates a labeled dbxref editor
    • SYNONYM_EDITOR - Creates a labeled synonym editor
    • CATEGORY_EDITOR - Creates a labeled category editor
    • TEXT_COMMIT - Creates a commit button
    • TEXT_REVERT - Creates a revert button
  • FIND - Creates a  Search/Filter/Render control
  • SEARCH_BUTTON - Creates a custom search button. See Custom Search Buttons for information on how to configure the many layout parameters of this components.
