How Do I Find Terms Without an is_a Parent?

There are two ways to do this:

Method 1: Use the "Has isa parent" filter

  1. Open a Search Panel ([Search] -> [Search Panel]).
  2. Set up the filter as [Don't have] [Has is_a parent]
  3. Press Search. A list of terms with no is_a parent will be displayed. Note that the ontology roots -- having no parents at all -- will also appear in this list.

Method 2: Link Filtering

Set up a link filter to only only display is_a links. The filter can be set up in the Link Search Panel, or, in the Ontology Tree Editor and the Graph Editor, the quick filtering option can be used.

When a link filter is applied, any term without an is_a parent will appear as a root. This method has the advantage of being able to find terms without a parent of any specified type. For example, to find part_of orphans, just use part_of instead of OBO_REL:is_a in the link filter setup.

2a: Using the Link Filter Panel

  1. Open a Link Search Panel ([Search] -> [Link Search Panel]).
  2. Set up the filter as [Type] [Self] [ID] [equals] "OBO_REL:is_a"
  3. Click the Filter button.

2b: Using Quick Filtering

In an Ontology Tree Editor panel or Graph Editor panel, click the lowercase 'f' in the component toolbar. From the flyout menus, select "Show a single relationship" and then "is_a (OBO_REL:is_a)".
