Pubmed Ontology Generation Tool

The ontology generation process is structured in three major steps:

  1. Term Generation - Extract terminology from text
  2. Definition Generation - Find phrases suitable for natural language definitions for terminology
  3. Add to Ontology - Suggestions to add generated terms to the ontology
Example: Ontology generation for stem cells.
Load cell.obo available at http://obo.cvs.sourceforge.net/*checkout*/obo/obo/ontology/anatomy/cell_type/cell.obo

Step1: Term Generation

Query PubMed and extract the terminology
Submit the query "stem cell". The abstracts are retrieved from PubMed and terminilogy is extracted. Terms are listed by level of importance.

Figure: Candidate terms extracted from PubMed abstracts for query stem cell.


Search or Filter terms with regular expressions
The list of candidate terms can be searched and filtered using regular expression patterns (evaluated case insensitive).

  • Search selects the first term that matches the pattern and highlights its position in the table.
  • Filters hide candidate terms not matching the specified pattern.
    Examples of regular expressions:
    1. starts with 's' - ^s OR $\setminus$As
    2. starts with 's' or 't' - ^[st] OR $\setminus$A[st]
    3. ends with with 's' - s$ OR s$\setminus$Z
    4. 's' is optional - cells
    5. sequence (*) of arbitrary characters (.) - human .* cell

    Figure: Searching and selecting bone marrow.


    Figure: Filtering candidate terms that end with stem cell or stem cells.


    Select and add terms to clipboard
    Clicking the checkbox or hitting the spacebar on a row selects a term and adds it to the clipboard.

    Figure: Selecting candidate terms.


    Several queries for a topic can be issued, the selections are stored until the current session exists. For the stem cell example, the search for human stem cell gets more specific existing terminology

    Figure: Refine results by generating candidate terms for the query human stem cell



    Loading, saving clipboard and clipboard behaviour
    The clipboard content can be saved and loaded back from a file. Terms in the clipoard are enriched with synonyms and abbreviations found in later queries. All generated candidate terms matching terms labels from the clipboard are automatically selected and the merged representation is used then on.

    Figure: Saving clipboard content


    Figure: Tab delimited file containing the clipboard content.


    Step2: Definition Generation

    Generate Definiton Phrases
    Clicking the DEF icon generates the definitions table with phrases for the candidate term.

    Figure: Definition phrases for the term human mesenchymal stem cell.


    Definitions can be filtered as shown for candidate terms above. Selecting a definition phrase makes it appear in the editing area below the definition table along with known abbreviations found in the analyzed text.

    Figure: Generated phrases for term human mesenchymal stem cell.


    Figure: Edit definition for term human mesenchymal stem cell.


    Step3: Add to Ontology

    The generated and defined term can then be added to the ontology with the following options:

    1. Add as child of the selected term in the Ontology Tree Editor
    2. Search for a potential parent term
    3. Add as child of one of the parent terms of the currently selected term
    4. Find existing ontology terms found in the definition as potential parents
    5. Find similar existing terms from the ontology as potential parents

    Figure:Proposing suitable parent terms for the term human mesenchymal stem cell.
