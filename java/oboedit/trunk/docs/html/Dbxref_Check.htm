Dbxref Check

Until recently, OBO-Edit did not strictly enfore the OBO format restrictions on dbxrefs. Many ontologies now contain dbxrefs that are not OBO compatible. The Dbxref Check will display a warning if a dbxref is not OBO-compatible. It checks for:

  • Missing database identifiers - This includes dbxrefs with no characters before the colon (or dbxrefs with no colon at all)
  • Missing database identifiers - This includes dbxrefs with no characters after the colon
  • Non-URI characters - This includes dbxrefs with characters that are not defined as legal URI characters by the w3c. Legal URI characters are:
    • letters
    • digits
    • _ (underscore)
    • - (dash)
    • ! (exlamation point)
    • . (period)
    • ~ (tilde)
    • \ (backslash)
    • ' (single quote)
    • ( (open parenthesis)
    • ) (close parenthesis)
    • * (asterisk)
    • , (comma)
    • ; (semicolon)
    • # (pound/number sign)
    • : (colon)
    • $ (dollar sign)
    • & (ampersand)
    • + (plus sign)
    • = (equals sign)
    • ? (question mark)
    • / (forward slash)
    • [ (open bracket)
    • ] (close bracket)
    • @ (at sign / arobase)
  • Invalid URLs - This includes dbxrefs that look like URLs, but are not valid urls. OBO-Edit thinks a dbxref "looks like a URL" if the dbxref database is http, https, ftp, telnet or ssh. OBO-Edit checks whether the URL is valid using the built-in Java URL syntax checker.
  • Description consitency - This includes dbxrefs that are used elsewhere in the ontology, but with a different description. OBO-Edit provides a quick fix that will change all matching dbxrefs to use the same description.
